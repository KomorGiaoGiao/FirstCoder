# FirstCoder 本地 Topic 边界分类实验设计

## 1. 目标与边界

本实验验证 `FacebookAI/xlm-roberta-base` 在 FirstCoder 场景中判断任务是否切换的效果。输入是当前任务上下文和最新用户消息，输出是 `same`、`new` 或基于概率阈值派生的 `uncertain`。

第一阶段只做离线数据处理、微调和评估，不替换或修改 FirstCoder 当前的隐藏 LLM 分类器，不改变运行时行为。原始 FirstCoder 与 Codex 日志全程留在本机，只读访问；模型下载、训练和推理也均在本机完成。

## 2. 已确认的数据来源

- FirstCoder：`~/.firstcoder/sessions/*.jsonl`。当前盘点到 146 个会话、593 条用户消息，其中 281 条非初始用户消息带现有分类器弱标签。
- Codex：`~/.codex/sessions/**/*.jsonl`。当前盘点到约 2,065 个日志文件、7,725 条真实用户消息。
- FirstCoder 的 `task_boundary_observed` 只作为预标注信号，不作为人工真值。
- Codex 的文件或任务边界不直接视为 Topic 边界；同一任务里可能切换 Topic，不同任务之间也可能续接旧 Topic。

两个来源分别解析，再归一到统一的中间格式。不得修改、移动或删除原始日志。

## 3. 样本定义

每个候选样本对应一条非会话首条的真实用户消息，至少包含：

```json
{
  "sample_id": "不可逆稳定 ID",
  "source": "firstcoder|codex",
  "session_group_id": "不可逆会话分组 ID",
  "context": "脱敏后的当前任务上下文",
  "new_message": "脱敏后的新用户消息",
  "weak_label": "same|new|uncertain|null",
  "human_label": "same|new|uncertain|null",
  "confidence": null,
  "review_reason": [],
  "split": "train|validation|test"
}
```

`context` 使用最新用户消息之前的有限有效上下文构造，保留用户与助手自然语言、当前目标、最近操作与验证状态；不直接塞入完整工具输出。最终模型输入按 tokenizer 截断到最多 512 tokens，优先保留新消息及距离它最近的上下文。

标签规则：

- `same`：新消息自然延续当前目标、交付物或未完成状态，包括短指代、验证、补充与修正。
- `new`：新消息开启不同目标、主题、交付物或问题，即使涉及相同文件或技术名词。
- `uncertain`：上下文不足或存在两个同样合理的解释。它是人工复核与推理兜底状态，不作为模糊样本强行训练进二分类头。

## 4. 脱敏与本地安全

数据提取阶段对以下内容做确定性脱敏：

- API Key、Bearer Token、Cookie、Authorization 头和常见凭证格式；
- 邮箱、手机号、IP 地址及带用户名的绝对主目录；
- `.env` 值、私钥块、认证配置和工具输出中的秘密；
- 附件正文及大段文件内容，仅保留必要的类型或占位符；
- 会话 ID、消息 ID 和原始路径改为不可逆哈希或稳定占位符。

脱敏后的候选集、标注集、模型权重和训练产物放在 Git 忽略的本地实验目录中。任何抽样检查默认只输出统计值、哈希 ID 和脱敏文本，不打印原始秘密。脚本测试使用合成日志 fixture，不读取真实用户日志。

## 5. 500 条种子集与复核策略

第一轮建立 500 条高质量种子样本，而不是从全部候选中纯随机抽样：

- 平衡 `same` 与 `new` 候选，避免模型只学多数类；
- 同时覆盖 FirstCoder 与 Codex；
- 提高困难短消息的占比，如“继续”“就这个”“跑一下”“刚才那个”；
- 覆盖显式切换表达，如“另外”“换个问题”“顺便”；
- 覆盖同技术名词但目标改变、不同词语但目标延续等难负例和难正例；
- 去除近重复样本，防止模板化消息主导训练。

初标由完整脱敏上下文和可用弱标签共同产生。用户不需要逐条复核 500 条：主要复核低置信度、来源信号冲突、上下文不足和规则命中的争议样本；另从高置信度样本中随机抽查一批，估计初标偏差。

测试集保留 100 条，要求全部人工确认；其余样本用于训练和验证。数据按完整 `session_group_id` 划分，禁止同一会话跨训练、验证和测试集，避免上下文泄漏。

## 6. 模型与训练

模型从 Hugging Face 下载 `FacebookAI/xlm-roberta-base` 到本地、Git 忽略的模型目录。使用 `AutoModelForSequenceClassification(num_labels=2)` 微调 `same/new`，不从零训练。

M1 16 GB 初始参数采用保守配置：最大长度 512、较小 batch、梯度累积、MPS 可用时启用 MPS。训练配置、随机种子、模型 revision、数据集 manifest 哈希和软件版本必须写入运行产物，以便复现。

模型先输出 `P(same)` 与 `P(new)`。离线评估再根据验证集选择双阈值：高于 `new` 阈值输出 `new`，高于 `same` 阈值输出 `same`，中间区域输出 `uncertain`。阈值选择优先降低 `new` 被误判为 `same` 的风险。

## 7. 评估与成功判据

核心报告包括：

- 独立测试集准确率、宏平均 F1；
- `new` precision、recall、F1；
- 最重要的 `new -> same` 错误数量与比例；
- `uncertain` 覆盖率，以及排除 `uncertain` 后的覆盖准确率；
- 混淆矩阵；
- 按来源、消息长度和困难类型分组的指标；
- M1 本地单条和批量推理延迟；
- 脱敏后的代表性错例及错误原因。

实验同时保留两个基线：始终预测多数类的朴素基线，以及 FirstCoder 已有弱标签在人工确认测试子集上的表现。第一轮不宣称可替换运行时分类器，只回答“本地 XLM-R 在当前数据规模下达到什么效果、主要错在哪里、增加哪些数据最有价值”。

## 8. 组件与数据流

实现应拆成可独立测试的单元：

1. FirstCoder 日志适配器：解析用户消息、前序上下文和弱标签关联。
2. Codex 日志适配器：解析真实用户轮次并排除开发者指令、系统注入和工具事件。
3. 脱敏器：对文本和元数据做确定性替换，并输出命中统计。
4. 样本构造器：生成上下文文本对、稳定 ID、质量标记和来源元数据。
5. 抽样与标注导出器：生成 500 条种子集和人工复核文件。
6. 训练器：下载或加载本地 XLM-R，完成可复现微调。
7. 评估器：调阈值并生成 JSON 与 Markdown 报告。

数据流为：

```text
只读原始日志
  -> 来源适配器
  -> 统一候选样本
  -> 脱敏与质量过滤
  -> 分组抽样和初标
  -> 人工复核
  -> 按会话切分
  -> XLM-R 微调
  -> 独立测试与错例报告
```

## 9. 错误处理与可恢复性

- 单个损坏 JSONL 文件不得终止整批处理；记录文件哈希、行号和错误类别后继续。
- 缺失消息锚点、缺失前文或无法脱敏确认的样本进入拒绝清单，不进入训练集。
- 下载中断可安全重试，不覆盖已验证的完整文件。
- 每个阶段写独立 manifest，后续阶段只消费上一步已校验产物，避免半成品混入。
- 脚本默认 dry-run 并显示计数；真实导出必须显式指定输出目录。

## 10. 明确不做

- 不修改或替换 `firstcoder/agent/task_boundary_classifier.py`。
- 不把模型接入 TUI、AgentLoop 或 session 状态机。
- 不用云端 LLM 接收原始或脱敏日志正文。
- 不把公开 Topic Segmentation 数据混入第一轮训练。
- 不训练大语言模型，不从零预训练 XLM-R。
- 不以弱标签指标代替人工确认测试集结果。
